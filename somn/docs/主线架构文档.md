## 二、主线调用链

### 2.1 串联运转（Serial）— 步骤链式执行

**适用场景**：增长策略生成、需求分析、漏斗优化等按顺序执行的任务

```
AgentCore.process_input()
  └─► SomnCore.process_with_wisdom()
        └─► SuperWisdomCoordinator.coordinate()
              └─► WisdomDispatcher.dispatch()        ← 串联第1步
                    └─► [各学派智慧依次调用]
                          RU_WISDOM → DAO_WISDOM → BUDDHA_WISDOM → ...
                    └─► 返回融合决策
              └─► GlobalWisdomScheduler.schedule()   ← 串联第2步
                    └─► [调度23个学派依次响应]
              └─► ThinkingMethodFusionEngine.fuse()  ← 串联第3步
                    └─► [思维方法依次融合]
  └─► 返回最终响应
```

### 2.2 并形运转（Parallel）— 多路并发执行

**适用场景**：深度推理、多角度分析、批量评估等可并行处理的任务

```
AgentCore.process_input()
  └─► SomnCore.process_with_wisdom()
        └─► SuperWisdomCoordinator.coordinate()
              └─► ParallelRouter.route([              ← 并形入口
                    │ wisdom_dispatcher,              ← 路1: 智慧调度
                    │ deep_reasoning_engine,          ← 路2: 深度推理
                    │ neural_scheduler,               ← 路3: 神经调度
                    │ learning_coordinator,           ← 路4: 学习协调
                    │ autonomous_agent                ← 路5: 自主智能
                  ])
                    │
                    ├──► WisdomDispatcher.dispatch()  [并发]
                    ├──► DeepReasoningEngine.reason() [并发]
                    ├──► Tier3NeuralScheduler.schedule() [并发]
                    ├──► LearningCoordinator.learn()  [并发]
                    └──► AutonomousAgent.run()        [并发]
                    │
                    ▼
              └─► ResultAggregator.gather()         ← 结果聚合
  └─► 返回综合响应
```

### 2.3 交叉运转（Cross）— 网状交叉影响

**适用场景**：元推理、反思优化、策略迭代等需要多模块相互反馈的任务

```
AgentCore.process_input()
  └─► SomnCore.process_with_wisdom()
        └─► SuperWisdomCoordinator.coordinate()
              └─► MetaSynthesisEngine.cross_weave()  ← 交叉入口
                    │
                    ├──► WisdomDispatcher ──────┐
                    │   [输出] ────────────────┼──► [反馈] ─► 神经记忆
                    ├──► ReasoningEngine ──────┤
                    │   [输出] ────────────────┼──► [反馈] ─► 学习系统
                    ├──► NeuralScheduler ──────┤
                    │   [输出] ────────────────┼──► [反馈] ─► 价值系统
                    ├──► LearningSystem ───────┤
                    │   [输出] ────────────────┼──► [反馈] ─► 目标系统
                    └──► AutonomousAgent ──────┘
                          [输出] ───────────────► [反馈] ─► 反思引擎
                    │
                    ▼
              └─► FeedbackLoop.process()           ← 反馈闭环
  └─► 返回进化后响应
```

## 四、主线数据流

### 4.1 请求上下文（Context）在主线上的传递

```
user_input → AgentCore
  ↓
context = {
    "user_input": str,
    "user_id": str,
    "session_id": str,
    "task_type": str,        # chat / growth_plan / demand_analysis / ...
    "industry": str,
    "wisdom_mode": str,      # serial / parallel / cross
    "depth": str,            # shallow / normal / deep
    "trace": [],             # 执行轨迹（用于追踪）
    "feedback": {},          # 用户反馈
    "memory": {},            # 记忆上下文
    "q_values": {},          # Q值学习数据
    "roi_data": {}           # ROI追踪数据
}
  ↓
SomnCore.process_with_wisdom(context)
  ↓ 传递到所有节点
SuperWisdomCoordinator → GlobalWisdomScheduler → WisdomDispatcher → 各学派智慧
  ↓
DeepReasoningEngine → NarrativeIntelligenceEngine → Tier3NeuralScheduler
  ↓
LearningCoordinator → AutonomousAgent → NeuralMemorySystem
  ↓
返回结果 + 更新后的 context
  ↓
AgentCore → 最终响应
```

### 4.2 三种运行模式的上下文切换

```python
# 串联模式：context["wisdom_mode"] = "serial"
# → 按顺序执行：WisdomDispatcher → GlobalWisdomScheduler → ThinkingMethodFusion

# 并形模式：context["wisdom_mode"] = "parallel"
# → 并发执行：threading.Thread / asyncio.gather

# 交叉模式：context["wisdom_mode"] = "cross"
# → 网状执行：FeedbackLoop 驱动多节点交叉影响
```

### 8.4 三种运行模式实现

**串联运转（Serial）**：
- 适用：WisdomDispatcher → GlobalScheduler → ThinkingEngine
- 执行方式：顺序链式调用

**并形运转（Parallel）**：
- 适用：DeepReasoning / NarrativeEngine / NeuralScheduler / LearningCoord / AutonomousAgent
- 执行方式：ThreadPoolExecutor / asyncio.gather

**交叉运转（Cross）**：
- 适用：NeuralMemory → ReinforcementTrigger → ROITracker → FeedbackLoop
- 执行方式：FeedbackLoop 驱动多节点交叉影响

#### ParallelRouter 并形路由（v1.3 新增）

**文件**: `src/main_chain/parallel_router.py`

**核心功能**:
- 多路并发路由：将任务同时分发到多个模块并发执行
- 智能策略选择：支持 ALL / BEST_K / BALANCED / DOMAIN_MATCH
- 结果聚合与冲突处理：合并多模块输出，处理战略方向冲突
- 容错降级：部分失败时自动降级

**使用方式**:
```python
from src.main_chain.parallel_router import ParallelRouter, get_parallel_router

router = get_parallel_router()
result = router.execute_main_chain_parallel(context)
```

**主线定位**:
- 负责并形运转模式
- 协调 DeepReasoning / NarrativeEngine / NeuralScheduler / LearningCoord / AutonomousAgent

#### CrossWeaver 交叉织网（v1.3 新增）

**文件**: `src/main_chain/cross_weaver.py`

**核心功能**:
- 网状交叉影响：构建模块间 27+ 条交叉链接
- 跨模块信号传递：支持 POSITIVE / NEGATIVE / NEUTRAL / ADAPTIVE 四种反馈
- 多跳传播：信号可沿链路多跳传播
- 进化驱动：基于反馈驱动系统进化

**交叉网络结构**:
```
wisdom_dispatcher → deep_reasoning → autonomous_agent → roi_tracker
       ↓                    ↓              ↓             ↓
learning_coordinator → semantic_memory → reflection_engine → goal_system
       ↓                    ↓              ↓             ↓
neural_memory ← reinforcement_trigger ← narrative_engine
```

**使用方式**:
```python

weaver = get_cross_weaver()
result = weaver.execute_main_chain_cross(initial_output, context)
```

**文件**: `src/main_chain/main_chain_integration.py`

**核心功能**:
- 统一入口：协调 ParallelRouter 和 CrossWeaver
- 动态模式选择：基于任务复杂度自动选择运行模式
- 上下文管理：`ChainContext` 贯穿整个执行链路
- 结果追踪：`ChainExecutionResult` 记录执行结果

**模式决策规则**:
| 条件 | 模式 |
|------|------|
| 复杂度 >= 0.7 或多维度关键词 | PARALLEL |
| 反思/反馈关键词 | CROSS |
| 复杂度 >= 0.5 | SERIAL |
| 路由为 orchestrator 且复杂度 < 0.3 | SERIAL |

**SomnCore 接入位置**:

requirement["parallel_insights"] = parallel_output.get("insights", [])

# 交叉模式: 在评估前执行交叉反馈

## 九、全局配置说明（v1.7 新增）

### 9.1 配置文件

**路径**: `config/main_chain_config.yaml`

**配置结构**:
```yaml
# 基础配置
main_chain_enabled: true           # 全局启用/禁用
default_mode: auto                  # 默认运行模式

# 并形路由器配置
parallel_router:
  max_concurrent: 5                # 最大并发数
  timeout: 30.0                    # 任务超时（秒）
  strategy: best_k                 # 路由策略
  mode: auto                       # 执行模式

# 交叉织网器配置
cross_weaver:
  max_iterations: 3               # 最大迭代次数
  max_hops: 3                     # 信号传播最大跳数

# 监控器配置
main_chain_monitor:
  max_history: 100                # 历史记录数
  sampling_interval: 60           # 采样间隔（秒）

# 模式决策配置
mode_decision:
  complexity:
    parallel_threshold: 0.7        # 并行模式复杂度阈值
    serial_threshold: 0.5          # 串行模式复杂度阈值
```
