## 🏗️ 系统架构

### 三层学习模型

```
Daily Learning Report (日报告)
    ↓
┌─────────────────────────────────────────────────┐
│                  三层学习执行                      │
├─────────────────────────────────────────────────┤
│                                                 │
│ 1️⃣ 本地学习层 (Local Learning)                  │
│    ├─ 职责: 扫描和学习本地数据                    │
│    ├─ 执行: 有数据则学习，无数据则跳过            │
│    ├─ 质量: >= 0.8                              │
│    └─ 阈值: >= 1条数据                          │
│                                                 │
│ 2️⃣ 网络学习层 (Network Learning)               │
│    ├─ 职责: 执行网络研究（必须）                 │
│    ├─ 执行: 每日必须，不受其他条件影响            │
│    ├─ 质量: >= 0.7                              │
│    └─ 主题: 5个默认研究方向                      │
│                                                 │
│ 3️⃣ 交叉融合层 (Cross-Learning)                 │
│    ├─ 职责: 融合两层数据生成洞察                 │
│    ├─ 执行: 两层都有数据时触发                   │
│    ├─ 分析: 关联发现 → 模式提取 → 洞察生成      │
│    └─ 输出: 高价值综合洞察                       │
│                                                 │
└─────────────────────────────────────────────────┘
    ↓
报告生成和保存
```

## 🎯 核心特性

### 特性1: 本地学习可选执行
```
如果本地有新数据(24h内) → 执行本地学习
如果本地无新数据 → 跳过本地学习

判断逻辑：
    data_count >= local_data_threshold
    ├─ YES → 执行 ✅
    └─ NO → 跳过 ⊘
```

**优势**:
- 充分利用内部数据
- 节约计算资源
- 灵活自适应

### 特性2: 网络学习必须执行
```
每日必须执行网络学习，不受其他条件影响

预期执行：
    ├─ 即使本地无数据 → 网络照样执行
    ├─ 即使系统刚启动 → 网络照样执行
    └─ 保证知识的连续性和完整性
```

**优势**:
- 持续学习行业趋势
- 保证知识丰富性
- 不会因无本地数据而停止学习

### 特性3: 交叉融合条件触发
```
当两层都有数据时自动触发交叉融合

条件：
    IF local_data AND network_data
        ├─ 发现关联 (两层数据的对应关系)
        ├─ 提取模式 (跨数据源的一致性)
        └─ 生成洞察 (高价值综合发现)
    ELSE
        └─ 跳过交叉融合
```

**优势**:
- 生成高价值洞察
- 两层数据交叉验证
- 提升发现的可信度

## 📊 执行流程

### 日报告执行流程图

```
09:00 触发
  ↓
扫描本地数据 (24h内)
  ├─ 统计数据条数
  ├─ 评估数据质量
  └─ 记录扫描结果
  ↓
判断本地学习
  ├─ IF count >= threshold
  │  └─ 执行本地学习 ✅
  └─ ELSE
     └─ 跳过本地学习 ⊘
  ↓
执行网络学习 (必须)
  ├─ 扫描研究主题
  ├─ 获取网络数据
  └─ 生成网络发现 ✅
  ↓
判断交叉条件
  ├─ IF local + network both exist
  │  └─ 执行交叉融合 ✅
  │     ├─ 发现关联
  │     ├─ 提取模式
  │     └─ 生成洞察
  └─ ELSE
     └─ 跳过交叉融合 ⊘
  ↓
生成日报告
  └─ DLR_YYYY-MM-DD.yaml
  ↓
保存报告
  └─ data/learning/reports/
```

```python
from src.neural_memory.three_tier_learning import run_daily_three_tier_learning
```

### 详细控制
```python
from src.neural_memory.three_tier_learning import ThreeTierLearningExecutor

executor = ThreeTierLearningExecutor()

# 执行本地学习
local_task = executor.execute_local_learning()
print(f"本地学习: {local_task.status.value}")

# 执行网络学习
network_task = executor.execute_network_learning()
print(f"网络学习: {network_task.status.value}")

# 执行交叉融合
cross_task = executor.execute_cross_learning(
    local_data=local_task.findings,
    network_data=network_task.findings
)

# 完整日报告
report = executor.execute_daily_learning()
print(report.summary['execution_flow'])
```

```python
executor.cross_trigger_threshold = 2   # 修改交叉触发阈值
report = executor.execute_daily_learning()
```

## 🔧 配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `local_data_threshold` | 1 | 本地数据≥此值时执行本地学习 |
| `network_data_threshold` | 1 | 网络学习执行阈值（通常不变） |
| `cross_trigger_threshold` | 1 | 交叉融合的数据触发数 |

