# 解决方案引擎 V2 - 动态评估架构

## 概述

解决方案引擎 V2 是基于**客户实际情况的动态效果评估系统**，彻底改变了 V1 版本中"固定预期效果"的局限性。

### 核心改进

| 维度 | V1 (固定模板) | V2 (动态评估) |
|------|--------------|---------------|
| 预期效果 | 固定数值 (如: CAC降低20-40%) | 基于客户参数动态计算 |
| 核心目标 | 预设目标 | 基于品牌/产品经营情况综合评估 |
| 关键指标 | 固定指标 | 根据行业和场景调整 |
| 执行能力 | 不考虑 | 核心因子 (70%实现率) |
| 行业差异 | 粗略分类 | 细分行业基准 |
| 评估验证 | 无 | 80%-120%阈值检测 |

### 2. 核心目标基于综合评估

**核心原则**: 核心目标不是基于痛点制定，而是基于**品牌、产品的实际经营情况综合评估**得出。

**评估维度**:
- **行业特性**: 不同行业的增长逻辑和关键驱动因素
- **品牌发展阶段**: 初创期/成长期/成熟期的目标差异
- **历史经营数据**: 增长率、客单价、复购率等基础指标
- **竞争格局**: 市场饱和度、竞争强度
- **资源禀赋**: 团队规模、资金实力、执行能力
- **痛点匹配度**: 解决方案与痛点的契合程度

```python
# 经营健康度评分 (1-10)
health_score = calculate_business_health(
```

### 3. 执行能力现实校准

**核心观点**: 不同行业、不同品牌、不同场景的预期效果完全不同。

```python

```

### 2. 执行能力现实校准

**核心原则**: 专业执行团队最多能达到理想状态的70%。

```python
# 执行能力评分 1-10 分
# 映射到实现率: 50% - 90%
# 公式: 实现率 = 0.5 + (评分-1)/9 * 0.4

execution_factor = 0.5 + (execution_score - 1) / 9 * 0.4

# 示例:
# 执行能力 4分 -> 实现率 63%
# 执行能力 6分 -> 实现率 72%
```

### 4. 评估准确性验证

**验证规则**:
- 实现率 < 80%: 评估过高 (over_estimated)
- 实现率 > 120%: 评估过低 (under_estimated)
- 80% - 120%: 评估合理 (valid)

```python
# 计算相对于行业基准的比例
```

### 核心组件

#### 1. SolutionTemplateV2 (动态模板)

```python
@dataclass
class SolutionTemplateV2:
    solution_id: str
    solution_type: SolutionType
    name: str
    
    # 评估参数定义 (客户需要提供的信息)
    required_parameters: List[AssessmentParameter]
    
    # 动态指标定义 (预期效果)
    dynamic_metrics: List[DynamicMetric]
```

#### 2. AssessmentParameter (评估参数)

```python
@dataclass
class AssessmentParameter:
    param_name: str           # 参数名称
    param_type: str           # 类型: float/int/str/bool/list
    description: str          # 描述
    required: bool = True     # 是否必填
    default_value: Any = None # 默认值
    valid_range: Tuple = None # 有效范围
```

**示例参数**:
- `industry`: 所属行业 (必填)
- `avg_order_value`: 平均客单价 (必填)
- `execution_capability`: 执行能力评分 1-10 (必填)
- `pain_points`: 主要痛点列表 (必填)
- `current_cac`: 当前获客成本 (选填)
- `team_size`: 团队规模 (选填)

#### 3. DynamicMetric (动态指标)

```python
@dataclass
class DynamicMetric:
    metric_id: str
    metric_name: str
    unit: str
    
    # 按行业细分的基准区间
    industry_baselines: Dict[str, Dict]
    
    # 影响因素权重
    factor_weights: Dict[str, float]
```

**示例指标**:
```python
DynamicMetric(
    metric_id="cac_reduction",
    metric_name="获客成本降低",
    unit="%",
    industry_baselines={
        "美妆": {"low": 20, "typical": 35, "high": 55},
        "母婴": {"low": 15, "typical": 30, "high": 45},
        "零售": {"low": 10, "typical": 25, "high": 40},
    },
    factor_weights={
        "execution_capability": 0.3,
        "team_size": 0.2,
        "industry": 0.25,
        "pain_point_match": 0.25
    }
)
```

#### 4. DynamicOutcomeCalculator (效果计算器)

计算公式:
```
最终效果 = 行业基准 × 调整因子 × 执行能力因子

其中:
- 行业基准: 根据行业选择的 low/typical/high
- 调整因子: 各影响因子的加权平均
- 执行能力因子: 0.5-0.9 (对应1-10分)
```

## 评估因子详解

### 1. 执行能力因子 (最重要)

```python
def _calculate_execution_factor(self, execution_score: float) -> float:
    """
    执行能力 1-10 分映射到实现率 50%-90%
    
    评分标准:
    1-3分: 执行能力弱，缺乏经验
    4-6分: 执行能力中等，有基础经验
    7-8分: 执行能力强，有成功案例
    9-10分: 执行能力极强，行业领先
    """
    return 0.5 + (execution_score - 1) / 9 * 0.4
```

### 3. 痛点匹配因子

```python
def _calculate_pain_point_factor(self, context, solution_type) -> float:
    """
    解决方案与痛点的匹配度
    """
    solution_pain_mapping = {
        "private_domain": [
            PainPointType.HIGH_CAC,      # 获客成本高

        ],
        "membership": [
            PainPointType.LOW_RETENTION,  # 留存率低
            PainPointType.LOW_CONVERSION  # 转化率低
        ]
    }
    
    # 匹配度越高，效果越好
    matched_pains = [p for p in context.primary_pain_points 
                     if p in relevant_pains]
    match_ratio = len(matched_pains) / len(relevant_pains)
    return 0.9 + (match_ratio * 0.3)  # 0.9 - 1.2
```

### 5. 历史表现因子

```python
def _calculate_historical_factor(self, context) -> float:
    """
    历史增长率反映潜力
    """
    historical_growth = context.historical_growth_rate
    
    if historical_growth < 0:
        return 0.85   # 负增长，保守估计
    elif historical_growth < 20:
        return 0.95   # 低增长
    elif historical_growth < 50:
        return 1.05   # 中速增长
    else:
        return 1.15   # 高速增长，潜力大
```

## 总结

解决方案引擎 V2 的核心价值:

1. **告别一刀切**: 每个客户都有定制化的预期效果
2. **现实校准**: 考虑执行能力现实，避免过度承诺
3. **数据驱动**: 基于行业基准和实际案例
4. **持续学习**: 从服务商案例中不断更新基准数据
5. **质量保障**: 80%-120%阈值确保评估准确性

使用 V2 引擎，您可以为每个客户提供更准确、更可信的解决方案评估。
