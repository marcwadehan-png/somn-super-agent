> 最后更新：2026-04-05  
> 覆盖范围：入口 → 核心调用链 → 子系统 → 反馈闭环

## 一、系统入口（3 个）

| 入口 | 文件 | 触发方式 | 说明 |
|---|---|---|---|
| **GUI 入口** | `main.py` → `src/gui/main_window.py` | `python main.py` | PySide6 桌面应用，`MainWindow._on_send_message()` → `AgentWorker.run()` → `agent_core.process_input()` |
| **CLI 入口** | `somn_cli.py` | `python somn_cli.py [-q "问题"]` | 命令行交互/单次查询，`create_agent()` 创建 AgentCore 实例后调用 `agent.process_input()` |
| **模块入口** | `src/somn.py`（Somn 类） | 程序化调用 `Somn.analyze(AnalysisRequest)` | 独立的增长分析入口，5 层架构，支持 growth_plan / demand_analysis / funnel_analysis / journey_mapping / narrative_analysis |

**关键事实**：GUI 和 CLI 共用同一条 `AgentCore.process_input()` 路径。`src/somn.py` 是独立的增长引擎，不走 AgentCore。

### 2.2 _handle_general_chat() 内部路由

这是最重要的处理器，覆盖论证型和任务型问题：

```
_handle_general_chat(user_input, context)     [agent_core.py:1078]
  │
  ├─ 0. _classify_input_intent() → intent_type
  │
  ├─ 0.5 intent_type == "conversational"
  │      └─→ _handle_conversational() → 直接自然回复
  │
  ├─ 1. intent_type == "argumentative" 且 somn_core 存在
  │      └─→ RAG 引擎增强上下文（v14.0.0）
  │          └─→ SuperWisdomCoordinator.analyze(query, context)
  │              └─→ 返回 primary_insight + secondary_insights + action_recommendations
  │
  ├─ 2. intent_type == "task" 或命中 complex_keywords
  │      └─→ _run_somn_chain() → SomnCore.run_agent_task()
  │          └─→ _extract_task_response() 提取策略/动作/评估/下一步
  │
  ├─ 3. 长文本(>4字) 备用论证
  │      └─→ SuperWisdomCoordinator.analyze() [第二次尝试]
  │
  └─ 4. 极短输入(≤4字) 兜底
         └─→ 情绪波动引擎(EmotionWaveEngine)修饰 → "说具体点,我好往下做."
```

### 3.2 任务路由决策（_assess_task_routing）

```
_assess_task_routing()  [somn_core.py:1803]

复杂度评估信号:
  - text_len_score × 0.15    (500字封顶)
  - action_score +0.3        (命中"帮我/给我/生成/制定/执行/完成/创建/分析/评估")
  - execution_score +0.3     (命中"执行/落地/实施/操作/完成/跑/运行")
  - deep_score +0.2          (命中"研究/深度/论证/报告/分析/策略/方案/全面")
  - constraint_score +0.05/个 (最多+0.2)
  - intent_bonus -0.1        (意图明确且为 question/chat)
  - intent_penalty -0.3      (意图为 chat/greeting/simple_question)

路由表:
  ┌────────────┬─────────────────┬──────────────────────────────┐
  │ 复杂度     │ 路由            │ 说明                         │
  ├────────────┼─────────────────┼──────────────────────────────┤
  │ < 0.35     │ orchestrator    │ 快手菜 - 本地模型快速响应     │
  │ 0.35~0.55  │ orchestrator    │ 家常菜 - 本地+云端老师辅助    │
  │ ≥ 0.55     │ full_workflow   │ 大餐 - 完整 strategy→执行→评估│
  └────────────┴─────────────────┴──────────────────────────────┘

用户显式指定优先: dining_mode / cuisine_mode / learning_mode
```

## 五、反馈闭环

### 5.1 PDCA 循环

```
执行阶段 (_record_workflow_feedback):
  P (Plan)  → 从 strategy_plan 提取计划节点
  D (Do)    → 记录每个任务的实际执行结果
  C (Check) → 对比 plan vs actual，记录偏差
  A (Act)   → 生成改进建议，回写到经验库

评估阶段 (_record_evaluation_learning_feedback):
  → FeedbackPipeline 接收反馈
  → ReinforcementTrigger 更新 Q 值
  → ROI 信号提取
```

### 5.2 学习闭环

```
用户交互 → AgentCore._learn_from_interaction()
  │
  ├─ MemorySystem.add_memory()       短期/长期记忆
  ├─ LearningSystem.record_session() 会话学习
  ├─ FeedbackPipeline.process()      反馈管道
  │    └─ ReinforcementTrigger       强化触发
  │         └─ Q 值更新 → 策略偏好调整
  └─ ROITracker.track()              ROI 追踪
       ├─ 任务级 ROI
       ├─ 工作流级 ROI
       └─ 策略组合级 ROI
```

### 5.3 自主进化闭环

```
经验库 (experience_library.json)
  ├─ _retrieve_similar_experiences() → TF-IDF 检索相似经验
  ├─ _store_autonomy_learning()      → 写回新经验
  ├─ 长期目标系统 (long_term_goals.json)
  │    └─ _upsert_long_term_goal()   → 目标创建/更新
  └─ 反思记录 (task_reflections.json)
       └─ _generate_autonomy_reflection() → 反思生成
```

## 六、Persona 人设系统

```
Persona 引擎 (persona_core.py)
  │
  ├─ assess_signals(user_input)         信号评估
  │    ├─ 优先级: high / medium / low
  │    ├─ high → 脏话规劝 (可直接覆盖标准链路输出)
  │    └─ medium → Heal 治愈 (参考但不覆盖)
  │
  ├─ check_output_style(user_input, response)  输出风格修饰
  │    └─ BriefMode: 简短化输出 (不拦截，只修饰)
  │
  └─ 9 种声线模式:
       CASUAL / SHARP / PROFESSIONAL / TENDER
       LATE_NIGHT / SOUL / FLIRTY / GUARDIAN / HEAL
```

**核心原则**：Persona 是"智能判断器/建议层"，不做硬拦截。标准链路始终执行。
