# 集成层设计文档

## 概述

集成层是Somn系统的核心协调组件，负责统一编排三大核心模块：
- **自主进化引擎** (Autonomous Evolution Engine)
- **行业知识图谱** (Industry Knowledge Graph)
- **AI原生增长策略** (AI-Native Growth Strategies)

## 架构设计

### 1. 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      UnifiedOrchestrator                     │

                       │

```

### 2. 核心组件

#### 2.1 IntegrationEventBus (事件总线)

**职责**：实现模块间的异步事件驱动通信

**核心功能**：
- 事件发布/订阅机制
- 优先级队列支持
- 事件历史记录
- 关联ID追踪

**事件类型**：
```python
class EventType(Enum):
    # 进化引擎事件
    EVOLUTION_STARTED/COMPLETED/FAILED
    HEALTH_STATUS_CHANGED
    ANOMALY_DETECTED
    
    # 知识图谱事件
    KNOWLEDGE_UPDATED
    ENTITY_ADDED
    RELATION_INFERRED
    
    # 增长策略事件
    STRATEGY_ACTIVATED
    EXPERIMENT_COMPLETED
    GROWTH_MILESTONE
```

#### 2.2 ModuleCoordinator (模块协调器)

**职责**：管理模块生命周期和依赖关系

**核心功能**：
- 模块注册与发现
- 依赖图管理（拓扑排序）
- 状态机管理
- 健康度监控

**依赖关系**：
```
Evolution Engine (无依赖)
       ↓
Knowledge Graph (依赖 Evolution Engine)
       ↓
Growth Strategies (依赖前两者)
```

#### 2.3 UnifiedOrchestrator (统一编排器)

**职责**：协调跨模块工作流和智能决策

**核心功能**：
- 工作流定义与执行
- 跨模块洞察生成
- 自动化触发规则
- 系统状态聚合

**工作流类型**：
1. **EVOLUTION_KNOWLEDGE_SYNC**: 进化结果同步到知识图谱
2. **KNOWLEDGE_GROWTH_RECOMMEND**: 基于知识生成增长策略
3. **FULL_CYCLE_OPTIMIZATION**: 全周期优化
4. **ANOMALY_RESPONSE**: 异常自动响应

#### 2.4 MonitoringSystem (监控告警)

**职责**：实时监控和告警管理

**核心功能**：
- 指标收集与存储
- 阈值规则引擎
- 告警生命周期管理
- 仪表盘数据聚合

**默认告警规则**：
- 系统健康度 < 60%: CRITICAL
- 系统健康度 < 80%: MEDIUM
- 模块故障: HIGH
- 工作流失败率 > 20%: HIGH

## 工作流程

### 1. 系统启动流程

```
1. 注册所有模块
2. 按依赖顺序初始化
   - Evolution Engine
   - Knowledge Graph
   - Growth Strategies
3. 启动事件总线
4. 启动监控系统
5. 启动编排器主循环
```

### 2. 典型工作流执行

#### 2.1 全周期优化工作流

```
Step 1: Evolution Engine
  └─ run_diagnostic()
       └─ 获取系统健康状态
       
Step 2: Knowledge Graph  
  └─ infer_all_relations()
       └─ 基于健康状态推理知识关系
       
Step 3: Growth Strategies
  └─ generate_optimization_plan()
       └─ 生成优化策略建议
       
Result: CrossModuleInsight
  └─ 整合三个模块的输出
```

#### 2.2 异常响应工作流

```
触发条件: ANOMALY_DETECTED 事件

Step 1: Evolution Engine
  └─ assess_anomaly()
       └─ 评估异常影响范围
       
Step 2: Knowledge Graph
  └─ find_related_cases()
       └─ 查询历史相似案例
       
Step 3: Growth Strategies
  └─ create_response_strategy()
       └─ 制定响应策略
```

### 3. 事件驱动机制

```
[Evolution Engine] ──EVOLUTION_COMPLETED──┐
                                           │
                                           ▼
[Event Bus] ──────────────────────────> [Orchestrator]
                                           │
                                           │ 触发工作流
                                           ▼
[Knowledge Graph] <──KNOWLEDGE_UPDATED────┘
```

## 配置指南

### 1. 阈值规则配置

```python
from integration.monitoring_system import ThresholdRule, AlertSeverity

rule = ThresholdRule(
    rule_id="custom_rule",
    metric_name="response_time",
    operator=">",
    threshold=1000,  # ms

async def custom_handler(context: Dict) -> Dict:

    priority=EventPriority.HIGH

```

## 性能考虑

### 1. 事件处理
- 使用异步处理避免阻塞
- 优先级队列确保关键事件优先
- 历史记录限制防止内存泄漏

### 2. 工作流执行
- 支持步骤并行执行（无依赖时）
- 超时机制防止死锁
- 结果缓存避免重复计算

### 3. 指标存储
- 使用deque限制历史大小
- 定期聚合减少内存占用
- 支持外部时序数据库集成

## 扩展性设计

### 1. 添加新模块

```python
# 1. 定义新模块类型
class ModuleType(Enum):
    NEW_MODULE = "new_module"

# 2. 更新依赖图
dependencies[ModuleType.NEW_MODULE] = {ModuleType.EVOLUTION_ENGINE}

# 3. 注册模块
coordinator.register_module(ModuleType.NEW_MODULE, instance, ...)
```

### 2. 添加新工作流类型

```python
# 1. 定义工作流类型
class WorkflowType(Enum):
    NEW_WORKFLOW = "new_workflow"

# 2. 实现处理器
async def handle_new_workflow(context):
    pass

# 3. 注册处理器
orchestrator._workflow_handlers[WorkflowType.NEW_WORKFLOW] = handle_new_workflow
```

## 故障处理

### 1. 模块故障
- 自动检测模块健康状态
- 依赖模块故障时暂停下游模块
- 支持模块热重启

### 2. 工作流失败
- 步骤级别错误捕获
- 支持工作流重试
- 失败原因记录和分析

### 3. 告警风暴
- 告警抑制机制
- 相似告警合并
- 升级策略配置
