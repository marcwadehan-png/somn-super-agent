## 1. 概述

本文档描述 Somn AI 项目的统一入口架构，实现"仅输入一条命令即可全局调用、一键运行并调动项目内所有代码完成全流程联动执行"的目标。

### 1.1 核心目标

- ✅ 单一入口: 通过 `run.py` 统一调用所有功能
- ✅ 多模式支持: GUI/CLI/单次查询/健康检查/状态查看
- ✅ 全模块打通: AgentCore → SomnCore → Intelligence 完整链路
- ✅ 一键启动: 通过 `start_somn.bat` / `start_somn.sh` 快速启动

## 3. 使用指南

### 3.1 快速启动

```bash
# Windows
.\start_somn.bat

# Linux/macOS
./start_somn.sh
```

### 3.2 运行模式

| 命令 | 功能 | 示例 |
|------|------|------|
| `python run.py` | 启动GUI (默认) | - |
| `python run.py --gui` | 启动GUI | - |
| `python run.py --cli` | CLI交互模式 | - |
| `python run.py --cli -q "查询"` | 单次查询 | `python run.py --cli -q "帮我制定增长策略"` |
| `python run.py --cli -q "查询" --industry xxx` | 带行业查询 | `python run.py --cli -q "分析市场" --industry saas_b2b` |
| `python run.py --health` | 健康检查 | - |
| `python run.py --status` | 系统状态 | - |
| `python run.py --solutions` | 列出解决方案 | - |
| `python run.py --test` | 模块测试 | - |
| `python run.py --version` | 版本信息 | - |

### 3.3 启动脚本用法

```bash

.\start_somn.bat --cli -q "帮我分析市场机会"

    pass
```

### 4.2 懒加载机制

```python
# AgentCore 的 Tier0 零同步开销初始化
class AgentCore:
    def __init__(self, memory_system, knowledge_base, config=None):
        # Tier 0: 零同步开销 - 全部组件占位为 None
        self.ml_engine = None
        self.somn_core = None
        self.persona = None
        # ... 其他组件
        
        # 启动后台预热 daemon
        self._bg_warmup_thread = threading.Thread(
            target=self._background_warmup,
            name="agent_bg_warmup",
            daemon=True,
        )
        self._bg_warmup_thread.start()
```

### 4.3 委托模式

```python
# SomnCore 的方法委托给子模块
class SomnCore:
    def analyze_requirement(self, description, context=None):
        """需求分析 - 委托给 _somn_main_chain"""
        from ._somn_main_chain import _module_run_analyze_requirement
        return _module_run_analyze_requirement(self, description, context)
    
    def design_strategy(self, requirement_doc, options=None):
        """策略设计 - 委托给 _somn_strategy_design"""
        from ._somn_strategy_design import run_design_strategy
        return run_design_strategy(self, requirement_doc, options)
```

## 7. 故障排除

### 7.1 常见问题

**Q: 启动时报 "Python未找到"**
- A: 确保Python已安装并添加到PATH环境变量

**Q: 启动时报 "依赖未安装"**
- A: 启动脚本会自动安装依赖，或手动运行 `pip install -r requirements.txt`

**Q: GUI模式无法启动**
- A: 确保已安装PySide6: `pip install PySide6`

**Q: 模块导入失败**
- A: 检查是否在项目根目录下运行，或尝试使用 `python run.py --test` 诊断

### 7.2 诊断命令

```bash
# 验证Python环境
python --version

# 验证依赖
python -c "import PySide6; print('PySide6 OK')"
python -c "from src.core.agent_core import AgentCore; print('AgentCore OK')"

# 运行模块测试
python run.py --test

# 查看系统状态
python run.py --status
```
