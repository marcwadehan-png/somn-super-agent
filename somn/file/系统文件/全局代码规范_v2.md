# Somn 全局代码规范 v1.0

**生效日期**: 2026-04-09  
**适用范围**: `smart_office_assistant/src/` 全部 Python 模块  
**规范性质**: 刚性要求，必须遵守

---

## 六项全局代码规范

### 原则一：只定义只读全局常量/配置，禁止可修改全局变量

**要求**:
- 允许定义 `Path`、`str`、`int`、`float`、`bool`、`tuple`、`frozenset` 等不可变类型的全局常量
- **禁止**在模块级别定义 `dict`、`list`、`set` 等可变类型的全局变量
- 如需可变容器，使用 `Final` 类型注解或惰性初始化函数

**合规示例**:
```python
from typing import Final

# ✅ 不可变常量
MAX_RETRIES: Final[int] = 3
DEFAULT_TIMEOUT: Final[float] = 30.0
CONFIG_PATH: Final[Path] = Path(__file__).parent / "config.yaml"
ALLOWED_MODES: Final[tuple] = ("auto", "manual", "test")

# ✅ 惰性初始化（推荐）
def get_empty_dict() -> dict:
    """返回空字典，保持向后兼容"""
    return {}
```

**违规示例**:
```python
# ❌ 禁止：模块级可变全局变量
results = []           # 禁止
cache = {}             # 禁止
config = {"debug": True}  # 禁止
```

### 原则二：仅做依赖导入、环境与全局单例资源一次性初始化

**要求**:
- 子模块**禁止**调用 `logging.basicConfig()`，仅使用 `logging.getLogger(__name__)` 获取 logger
- 主入口（`src/somn.py`）负责统一配置日志、初始化目录等全局资源
- 单例资源使用惰性初始化，避免模块导入时立即创建

**合规示例**:
```python
# src/core/my_module.py
import logging

# ✅ 仅获取logger，不配置
logger = logging.getLogger(__name__)

def do_something():
    logger.info("This is fine")
```

**违规示例**:
```python
# ❌ 禁止：模块级别配置日志
logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)
```

---

### 原则三：所有业务逻辑全部封装进函数/类，全局不写业务代码

**要求**:
- 模块级别**禁止**执行目录创建、文件写入、网络请求等副作用操作
- 模块级别**禁止**实例化引擎类或其他重型对象
- 所有业务逻辑必须封装在函数或类方法中

**合规示例**:
```python
# ✅ 惰性初始化
from functools import lru_cache

@lru_cache(maxsize=1)
def get_engine() -> "MyEngine":
    """获取引擎单例 - 首次调用时才创建"""
    return MyEngine()

# ✅ 目录初始化函数
def ensure_directories() -> None:
    """确保目录存在 - 由主入口调用"""
    DATA_DIR.mkdir(parents=True, exist_ok=True)
```

**违规示例**:
```python
# ❌ 禁止：模块级副作用
DATA_DIR.mkdir(parents=True, exist_ok=True)  # 禁止
engine = MyEngine()                           # 禁止
results = fetch_data()                        # 禁止
```

### 原则四：统一设置程序唯一入口，全局仅触发入口执行

**要求**:
- **唯一主入口**: `src/somn.py`
- **包级入口**: `src/__main__.py`（仅转发到主入口）
- **GUI入口**: `src/main.py`（集成到主入口逻辑）
- 其他所有模块**禁止**包含 `if __name__ == "__main__":` 代码块
- 测试入口统一移至 `tests/` 目录

**合规示例**:
```python
# src/somn.py - 唯一主入口
def main():
    setup_logging()
    ensure_directories()
    # ... 业务逻辑

if __name__ == "__main__":
    main()

# src/__main__.py - 包级入口
from src.somn import main
if __name__ == "__main__":
    main()
```

### 原则五：模块文件仅在全局做接口导出暴露，不做额外逻辑

**要求**:
- 所有模块必须定义 `__all__` 列表，明确公开 API
- `__all__` 应包含类、函数、类型别名等公开成员
- 内部实现（以下划线开头的成员）不应包含在 `__all__` 中

**合规示例**:
```python
# src/utils/config_manager.py

__all__ = [
    'ConfigManager',      # 公开类
    'get_config',         # 单例访问函数
    'ConfigType',         # 类型别名
]

class ConfigManager:
    ...

def get_config() -> ConfigManager:
    ...

# 内部实现，不导出
_config_manager = None
```

### 原则六（新增）：单个代码文件不得超过500行

**要求**:
- **硬性限制**: 单个 `.py` 文件不得超过 **500 行**（含注释和空行）
- 超过 500 行的文件必须进行拆分重构
- 拆分策略根据代码特点和功能重新规划

#### 拆分策略指南

| 代码特点 | 拆分策略 | 示例 |
|---------|---------|------|
| **大类多方法** | 拆分为包结构，方法按功能分组到子模块 | `somn_core.py` → `somn_core/` 包 |
| **多独立功能** | 每个功能一个模块，原文件作为兼容层 | `wisdom_dispatcher.py` → `_wd_*.py` |
| **工具函数集** | 按功能域分组，创建工具子包 | `utils.py` → `utils/` 子包 |
| **类型定义多** | 提取 `_types.py` 模块 | `*_types.py` |
| **常量配置多** | 提取 `_constants.py` 或 `config.py` | `*_constants.py` |

#### 拆分实施步骤

1. **扫描**: 识别超过500行的文件
2. **分析**: 分析代码结构，确定拆分边界
3. **搭建**: 创建包结构/子模块框架
4. **迁移**: 将代码迁移到子模块
5. **适配**: 原文件改为兼容层（保留导入兼容性）
6. **测试**: 验证拆分后功能正常

#### 合规示例

**拆分前** (`somn_core.py` - 1426行):
```
src/core/somn_core.py (1426行) ❌
```

**拆分后**:
```
src/core/somn_core/                    # 包目录
├── __init__.py                        # 兼容层，导出原API
├── _sc_base.py                        # 基础类定义
├── _sc_routes.py                      # 路由相关方法
├── _sc_analysis.py                    # 分析方法
├── _sc_strategy.py                    # 策略设计
├── _sc_execution.py                   # 工作流执行
├── _sc_evaluation.py                  # 评估方法
└── _sc_utils.py                       # 工具方法

src/core/somn_core.py (50行) ✅        # 兼容层，导入自包
```

#### 例外情况

以下情况可申请例外（需注释说明原因）：
- 自动生成的代码（如协议缓冲区、ORM模型）
- 第三方库的封装适配层
- 配置/数据文件（JSON/YAML/CSV）

## 违规处理流程

### 1. 扫描识别

```bash
# 扫描超过500行的文件
find src -name "*.py" -exec wc -l {} + | awk '$1 > 500 {print $0}'

# 扫描可变全局变量
grep -rn "^[A-Z][A-Z0-9_]*\s*=\s*\[\]" src/ --include="*.py"
grep -rn "^[A-Z][A-Z0-9_]*\s*=\s*{}" src/ --include="*.py"

# 扫描模块级日志配置
grep -rn "logging.basicConfig" src/ --include="*.py"

# 扫描多入口点
grep -rn "if __name__ ==" src/ --include="*.py"

# 扫描缺少 __all__ 的文件
for f in src/**/*.py; do 
    [ -z "$(grep "^__all__" "$f")" ] && echo "Missing __all__: $f"
done
```

### 2. 修复优先级

| 优先级 | 违规类型 | 修复时限 |
|-------|---------|---------|
| P0 | 原则一、二、三、四 | 本轮迭代内修复 |
| P1 | 原则五 | 下轮迭代内修复 |
| P2 | 原则六（>800行） | 两轮迭代内拆分 |
| P3 | 原则六（500-800行） | 持续拆分 |

### 3. 验证清单

```bash
# 1. 验证文件行数
find src -name "*.py" -exec wc -l {} + | awk '$1 > 500 {print $1, $2}'

# 2. 验证无模块级日志配置（除主入口外）
grep -rn "logging.basicConfig" src/ --include="*.py" | grep -v "somn.py" | grep -v "__main__"

# 3. 验证无模块级实例化
grep -rn "= .*Engine()" src/ --include="*.py" | grep -v "def " | grep -v "# "

# 4. 验证入口点唯一
grep -rn "if __name__ ==" src/ --include="*.py" | wc -l
# 期望: 1-3（somn.py、__main__.py、main.py）

# 5. 验证无可变全局常量
grep -rn "^[A-Z][A-Z0-9_]*\s*=\s*\[\]" src/ --include="*.py"
grep -rn "^[A-Z][A-Z0-9_]*\s*=\s*{}" src/ --include="*.py"

# 6. 验证核心模块有 __all__
for f in src/core/*.py src/utils/*.py; do
    if ! grep -q "^__all__" "$f"; then
        echo "Missing __all__: $f"
    fi
done
```

### C. 术语表

|-----|------|
| 全局状态 | 模块级别定义的变量，可在任何地方访问 |
