# Somn 存储架构设计文档 v1.0

## 📋 概述

本文档描述 Somn 智能助手的统一存储架构，包括运行数据库、日志数据库、核心数据存储和藏书阁备份系统。

## 🏗️ 架构概览

```
data/                          # 项目根目录下 data/
├── run/                       # 运行数据库
│   └── run.db                 # SQLite - 会话、任务、运行时数据
├── logs/                      # 日志数据库
│   └── logs.db               # SQLite - 操作、错误、性能、审计日志
├── core/                      # 核心数据
│   ├── config_backups/        # 配置备份
│   ├── snapshots/            # 状态快照
│   └── storage_reports/      # 存储报告
├── backups/                   # 临时备份
├── imperial_library/          # 藏书阁（Imperial Library）
│   ├── 甲/                   # 永久备份（永不删除）
│   ├── 乙/                   # 1年备份
│   ├── 丙/                   # 30天备份
│   ├── 丁/                   # 7天备份
│   └── library_index.json     # 备份索引
├── autonomous/                # 自主运行数据（运行时自动创建）
├── claws/                     # 776个Claw子智能体配置（运行时自动创建）
├── knowledge/                 # 知识库（运行时自动创建）
├── learning/                  # 学习数据（运行时自动创建）
├── memory/                    # 记忆数据（运行时自动创建）
└── research_phases/           # 研究阶段数据（运行时自动创建）
```

## 📊 数据库设计

### 运行数据库 (run.db)

| 表名 | 用途 | 主要字段 |
|------|------|----------|
| `sessions` | 会话记录 | session_id, start_time, end_time, user_id, status |
| `task_states` | 任务状态 | task_id, status, progress, result, error |
| `runtime_data` | 运行时数据 | key, value, value_type |
| `execution_history` | 执行历史 | timestamp, component, action, duration_ms, success |

### 日志数据库 (logs.db)

| 表名 | 用途 | 主要字段 | 保留策略 |
|------|------|----------|----------|
| `operation_logs` | 操作日志 | timestamp, level, component, action, message | 30天 |
| `error_logs` | 错误日志 | timestamp, error_type, error_message, stack_trace | 90天 |
| `performance_logs` | 性能日志 | timestamp, metric_name, metric_value, unit | 14天 |
| `audit_logs` | 审计日志 | timestamp, user_id, action, resource_type, result | 180天 |

## 🏛️ 藏书阁备份系统

### 备份级别

| 级别 | 符号 | 保留策略 | 数据类型 |
|------|------|----------|----------|
| 甲级 | 永久 | 永不删除 | 系统配置、智慧编码、学习模式、关键状态、ROI报告 |
| 乙级 | 年度 | 365天 | 会话摘要、任务历史、学习成果、研究成果 |
| 丙级 | 月度 | 30天 | 操作日志、性能摘要、错误报告 |
| 丁级 | 周度 | 7天 | 原始日志、临时数据 |

### 备份接口

```python
from smart_office_assistant.src.core.storage import (
    SomnStorageManager,
    BackupTier,
    get_storage_manager,
    backup_critical_data,
)

# 获取存储管理器
manager = get_storage_manager()

# 备份数据到藏书阁
backup_id = manager.backup_to_library(
    data={"key": "value"},
    category="test_data",
    tier=BackupTier.YI,  # 乙级
    title="测试备份",
    metadata={"source": "test"}
)

# 从藏书阁恢复
data = manager.restore_from_library(backup_id)

# 便捷函数：备份关键数据
backup_critical_data(
    data={"important": "data"},
    title="关键数据备份"
)
```

### 自动备份规则

| 数据类型 | 备份级别 | 触发条件 |
|----------|----------|----------|
| 系统配置 | 甲级 | 配置变更时 |
| 智慧编码 | 甲级 | 新编码生成时 |
| 学习结果 | 乙级 | 学习完成时 |
| ROI报告 | 甲级 | 报告生成时 |
| 会话摘要 | 乙级 | 会话结束时 |
| 任务结果 | 乙级 | 任务完成时 |
| 错误日志 | 丙级 | 错误发生时 |

## 📁 配置文件

### storage_config.yaml

位置：`config/storage_config.yaml`

主要配置项：

```yaml
paths:
  # 项目根目录由系统自动检测，无需手动配置
  project_root: ""
  run_dir: "data/run"
  log_dir: "data/logs"
  core_dir: "data/core"
  imperial_library: "data/imperial_library"

databases:
  run_db:
    path: "data/run/run.db"
    journal_mode: "WAL"
  log_db:
    path: "data/logs/logs.db"
    retention_days: 30

imperial_library_backup:
  enabled: true
  auto_backup:
    interval_minutes: 60
```

## 🔧 使用指南

### 初始化存储系统

```bash
cd /path/to/somn
python scripts/init_storage.py
```

### 存储管理器使用

```python
from smart_office_assistant.src.core.storage import get_storage_manager

manager = get_storage_manager()

# 会话管理
manager.create_session("session_001", user_id="user")
manager.end_session("session_001", context_summary="...")

# 任务状态
manager.update_task_state("task_001", "running", progress=0.5)
state = manager.get_task_state("task_001")

# 运行时数据
manager.set_runtime_data("config", {"setting": True})
value = manager.get_runtime_data("config")

# 日志记录
manager.log_operation("component", "action", "message")
manager.log_error("component", "ErrorType", "error message")
manager.log_performance("component", "latency", 125.5, "ms")

# 查询日志
logs = manager.query_logs("operation", level="INFO", limit=100)
```

### 藏书阁集成

```python
from smart_office_assistant.src.core.storage._library_integration import (
    get_library_integration,
    auto_backup_to_library,
)

integration = get_library_integration()

# 自动备份会话
integration.auto_backup_session(session_id, summary, session_data)

# 自动备份任务
integration.auto_backup_task(task_id, result)

# 自动备份智慧数据
integration.auto_backup_wisdom(wisdom_data, wisdom_type)

# 查看藏书阁状态
status = integration.get_library_status()
```

### 状态快照

```python
manager = get_storage_manager()

# 创建状态快照
snapshot_id = manager.create_state_snapshot(
    component="SomnCore",
    state={"mode": "auto", "active_tasks": 5}
)

# 加载快照
snapshot = manager.load_state_snapshot(snapshot_id)
```
