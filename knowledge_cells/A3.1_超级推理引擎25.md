# A3.1 超级推理引擎（25%+节点）

## 调度器信息
- 类型：深度推理调度引擎（轻量级）
- 触发条件：需要25%以上推理节点加载的问题
- 性能目标：3秒内完成基础深度推理

## 25%+节点加载配置

### 核心节点（8个）- 必加载
```
core_nodes = [
    "premise_extractor",    // 前提提取器
    "logic_validator",      // 逻辑验证器
    "reasoning_chain",      // 推理链构建
    "evidence_weight",      // 证据权重评估
    "conclusion_synthesizer", // 结论综合器
    "confidence_calculator", // 置信度计算
    "fallacy_checker",      // 谬误检查器
    "counterexample_finder"  // 反例发现器
]
```

### 扩展节点（10个）- 按需加载
```
extended_nodes = [
    "analogical_reasoner",  // 类比推理
    "causal_analyzer",      // 因果分析
    "counterfactual_engine", // 反事实推理
    "perspective_generator", // 视角生成
    "assumption_detector",  // 假设检测
    "logic_gap_finder",     // 逻辑缺口发现
    "evidence_chain",       // 证据链构建
    "meta_reasoning",       // 元推理
    "uncertainty_handler",  // 不确定性处理
    "explanation_generator" // 解释生成器
]
```

### 辅助节点（5个）- 可选加载
```
auxiliary_nodes = [
    "visualization_generator", // 可视化生成
    "reasoning_auditor",         // 推理审计
    "bias_detector",             // 偏差检测
    "knowledge_retriever",       // 知识检索
    "memory_updater"             // 记忆更新
]
```

## 调度入口

```
dispatch_super_reasoning(问题, node_load="25%")
  ├── initialize_nodes(core_nodes) → 初始化核心节点
  ├── load_extended_nodes(10) → 加载扩展节点
  ├── execute_reasoning_pipeline() → 执行推理管道
  │     ├── premise → logic → evidence → conclusion
  │     └── fallback_loops → 反馈优化
  └── return Result(confidence, explanation)
```

## 调度参数
- `node_load`: 节点加载比例（默认0.25，即25%）
- `reasoning_depth`: 推理深度（1-5）
- `time_limit`: 时间限制（默认3秒）
- `strict_mode`: 严格模式

## 与A3.2/A3.3的区别

| 版本 | 节点加载 | 翰林院审核 | 适用场景 |
