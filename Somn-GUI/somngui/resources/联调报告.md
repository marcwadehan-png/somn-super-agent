## 5. 架构验证

### 5.1 前后端分离确认

```
GUI (Somn-GUI)                    Backend (Somn)
┌─────────────────┐               ┌─────────────────┐
│  MainWindow     │               │  FastAPI Server  │
│  ↓              │               │  ↓              │
│  AppState       │──API──HTTP──→│  routes/         │
│  ↓              │               │  ↓              │
│  SomnAPIClient  │←──JSON───────│  deps.py         │
│  ↓              │               │  ↓              │
│  BackendConn.   │               │  SomnCore        │
└─────────────────┘               └─────────────────┘
```

- ✅ MainWindow 不直接导入 Somn 核心模块
- ✅ 全部通过 AppState → SomnAPIClient → HTTP REST → Backend
- ✅ 连接发现: 配置文件 → 注册文件(TEMP) → 默认地址探测

### 5.2 线程模型

| 组件 | 线程 |
|------|------|
| Qt GUI 主线程 | MainWindow, 所有 UI 更新 |
| ConnectionWorker (QThread) | 后端连接发现 |
| ApiWorker (QThread) | API 请求（每次请求新线程） |

## 6. 已知限制

1. **Qt Offscreen 模式** — Windows 下 `QT_QPA_PLATFORM=offscreen` 会触发 `STATUS_STACK_BUFFER_OVERRUN`，无法进行无头 GUI 渲染测试。GUI 窗口的实际显示需要在有桌面环境的机器上启动 `start_gui.py` 验证。
2. **对话功能** — 对话 API 依赖后端 LLM 初始化，当前返回空内容（后端已做优雅降级处理）。
3. **docx/pdf 导出** — 本地导出模块的 docx/pdf 依赖 `python-docx`/`reportlab`，尚未安装到前端 venv。
