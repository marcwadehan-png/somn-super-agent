## 方法论说明

### 5层深度分析法

#### 第一层: 理论层
- **目的**: 理解兵法/哲学的核心原理和思想体系
- **内容**: 战略思想、哲学基础、管理智慧
- **深度**: 从"是什么"到"为什么"

#### 第二层: 文本层
- **目的**: 分析具体计谋/篇章的内涵
- **内容**: 计谋原文、篇章要义、历史背景
- **深度**: 从"表面现象"到"内在逻辑"

#### 第三层: 知识层
- **目的**: 理解背后的智慧原理
- **内容**: 兵法原理、哲学思考、实践智慧
- **深度**: 从"文字表面"到"深层含义"

#### 第四层: 案例层
- **目的**: 通过具体案例理解抽象概念
- **内容**: 经典战例、历史典故、现代应用
- **深度**: 从"理论概念"到"具体情境"

#### 第五层: 应用层
- **目的**: 将思想应用到AI系统
- **内容**: 系统设计、策略引擎、决策支持
- **深度**: 从"理论理解"到"实践应用"

## 《三十六计》超深度解析

### 0. 作品概述

**《三十六计》**  
**成书时间**: 语源于南北朝，成书于明清  
**性质**: 中国古代兵法策略汇编  
**核心**: 三十六个军事、政治、外交策略  

**六大类别**:
1. **胜战计** (6计): 己方处于优势时的计谋
2. **敌战计** (6计): 与敌对峙时的计谋
3. **攻战计** (6计): 进攻作战时的计谋
4. **混战计** (6计): 混乱局势中的计谋
5. **并战计** (6计): 兼并作战时的计谋
6. **败战计** (6计): 己方处于劣势时的计谋

### 1. 胜战计 - 处于优势时的智慧

#### 1.1 第一计: 瞒天过海

**原文**:  
> 备周而意怠，常见则不疑，阴在阳之内，不在阳之对。太阳，太阴。

**超深度解析**:

**核心原理**:
- 防备周全时容易麻痹松懈
- 习以为常的事物不再引起怀疑
- 机密藏在公开的事物之中
- 非常公开的行为往往蕴藏非常机密

**哲学层次**:
- 阴阳相生: 阴中有阳，阳中有阴
- 矛盾转化: 常态→异常
- 逆向思维: 越公开越隐蔽

**历史案例**:
- 薛仁贵瞒天过海: 唐太宗渡海征辽，薛仁贵将船装饰成宫殿，秘密渡海
- 邓艾阴平渡险: 魏将邓艾偷渡阴平小道，奇袭成都

**现代应用**:
- 商业谈判: 声东击西，暗度陈仓
- 科技创新: 伪装研发方向，秘密突破
- 信息安全: 公开渠道传输机密信息

---

#### 1.2 第二计: 围魏救赵

**原文**:  
> 共敌不如分敌，敌阳不如敌阴。

**超深度解析**:

**核心原理**:
- 与其攻击集中的敌人，不如分散敌人
- 与其正面攻击，不如攻击敌人虚弱之处
- 避实击虚，以迂为直

**哲学层次**:
- 以逸待劳: 消耗敌人而非硬碰硬
- 釜底抽薪: 断其根本
- 舍近求远: 以迂为直，以患为利

**历史案例**:
- 孙膑围魏救赵: 攻其必救，歼其归路
- 曹操官渡之战: 烧乌巢粮草

**现代应用**:
- 商业竞争: 攻击对手核心业务
- 危机处理: 解决根本问题
- 谈判策略: 迂回谈判，达成目标

**超深度解析**:

**核心原理**:

**哲学层次**:
- 借势: 顺势而为
- 借力: 他山之石可以攻玉
- 借名: 挟天子以令诸侯

**历史案例**:
- 二桃杀三士: 晏子借刀杀人
- 苏秦合纵: 借六国之力抗秦

**现代应用**:
- 商业竞争: 利用媒体、监管打击对手
- 外交策略: 利用国际组织
- 职场博弈: 借力打力

#### 1.4 第四计: 以逸待劳

**历史案例**:
- 吴楚长岸之战: 养由基百步穿杨
- 陆逊夷陵之战: 以逸待劳，火烧连营

**现代应用**:

- 投资策略: 静观市场，等待机会
- 人生哲学: 厚积薄发

**现代应用**:

```python
class PublicFirstDecision:
    """贵公去私决策系统"""
    
    def evaluate(self, options):
        """评估选项的公私属性"""
        evaluations = []
        
        for option in options:
            public_benefit = self._calculate_public_benefit(option)
            private_cost = self._calculate_private_cost(option)
            
            score = public_benefit - private_cost
            evaluations.append({
                'option': option,
                'public_score': public_benefit,
                'private_cost': private_cost,
                'net_score': score
            })
        
        # 选择公利最大的选项
        return max(evaluations, key=lambda x: x['net_score'])
```
