## 体系总览

### 整合逻辑

六个研究体系分别覆盖研究全链路的不同环节，整合为统一的「五层金字塔」架构：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        第五层：策略演化与闭环                          │
│              [策略生命周期 · 关联网络 · ROI追踪 · 自进化]               │
│         ← 来源：研究发现驱动策略体系 + 核心策略速查手册 →              │
├─────────────────────────────────────────────────────────────────────┤
│                      第四层：场景动态适配与调度                        │
│            [9大场景 · V5网状调度 · 动态扩展 · 复杂度评估]              │
│         ← 来源：应用场景与策略动态体系 →                               │
├─────────────────────────────────────────────────────────────────────┤
│                     第三层：价值量化与数据驱动                         │
│          [价值发现 · 三维量化 · 维度生成 · ROI追踪 · OKR融合]         │
│         ← 来源：价值驱动数据研究体系 →                                 │
├─────────────────────────────────────────────────────────────────────┤
│                    第二层：交叉研究矩阵与理论                          │
│        [5维×6方向 30交叉点 · 神经科学 · AI情绪 · 跨文化]              │
│         ← 来源：消费情绪与感性决策交叉研究体系v2.0 →                   │
├─────────────────────────────────────────────────────────────────────┤
│                    第一层：研究基础设施与规范                          │
│           [4层级×6深度 · 方法库 · 质量门控 · AI增强 · 工具]            │
│         ← 来源：研究颗粒度层级体系v2.0 →                               │
└─────────────────────────────────────────────────────────────────────┘
```

**核心流转链路**:

```
研究需求
    ↓
第一层：选择层级(L1-L4) × 深度(D0-D5) × 方法 × 质量门控
    ↓
第二层：定位交叉研究点(5维×6方向)，调用理论框架与神经科学基础
    ↓
第三层：价值发现 → 三维量化评估 → 自动生成数据维度 → ROI预估
    ↓
第四层：识别应用场景 → V5复杂度评估 → 主线调度 → 跨线协同
    ↓
第五层：策略生成 → 验证 → 应用 → 成熟 → 衰退 → 退役 → 自进化
```

## 第一层：研究基础设施与规范

### 1.1 四大研究层级（固定维度）

| 层级 | 核心特征 | 适用场景 | 精度等级 | 成本等级 | 速度等级 |
|------|---------|---------|---------|---------|---------|
| **L1 定量研究** | 大规模、可量化、统计推断 | 验证假设、测量指标、发现规律 | 中-高 | 中 | 慢-中 |
| **L2 定性研究** | 深度、探索性、理解动机 | 探索未知、理解行为、挖掘需求 | 低-中 | 中 | 慢 |
| **L3 数字化工具** | 实时、全量、自动化 | 持续监测、预测分析、个性化 | 中 | 低-中 | 快 |
| **L4 神经科学** | 潜意识、生理指标、直接测量 | 验证本能反应、测量隐性偏好 | 极高 | 高 | 慢 |

**层级选择决策树**:

```
研究问题解析
    ↓
问题核心是什么？
    ├─ "有多少/比例/趋势？" ──→ L1 定量研究
    ├─ "为什么/怎么想/什么感受？" ──→ L2 定性研究
    ├─ "实时如何/预测/自动化？" ──→ L3 数字化工具
    ├─ "本能/潜意识/生理反应？" ──→ L4 神经科学
    └─ 复合问题（多个维度）──→ 跨层级组合（见§1.5）
```

### 1.2 六级研究深度（核心分级）

| 深度 | 名称 | 核心定义 | 典型耗时 | 输出要求 | 信息源数量 | 适用指令 |
|------|------|---------|---------|---------|----------|---------|
| **D0** | 快速扫描 | 信息检索+要点整理 | 5-30分钟 | 简要摘要(200-500字) | 1-3个 | "看看""查一下""了解" |
| **D1** | 基础研究 | 基本知识梳理+关键事实确认 | 1-4小时 | 结构化总结(500-1500字) | 3-8个 | "介绍一下""概述""整理" |
| **D2** | 标准研究 | 多源交叉验证+分析框架 | 4-16小时 | 研究简报(1500-3000字) | 8-20个 | "研究""分析""评估" |
| **D3** | 深入研究 | 文献综述+批判性分析+多视角 | 1-3天 | 研究报告(3000-6000字) | 20-50个 | "深入研究""全面分析" |
| **D4** | 博士研究 | 博士级深度+文献综述+理论框架 | 3-7天 | 学术级报告(6000-15000字) | 50-200个 | "博士级""学术研究" |
| **D5** | 论证研究 | 全网检索+多源论证+逻辑推导+博弈分析 | 3-14天 | 论证报告(10000-30000字) | 100-500+个 | "论证""严密论证" |

**深度判定规则**:

```
用户指令解析
    ↓
显式关键词匹配
    ├─ 包含"论证/严密论证" ──→ D5
    ├─ 包含"博士级/学术/论文级" ──→ D4
    ├─ 包含"深入/全面/彻底/系统" ──→ D3
    ├─ 包含"研究/分析/评估/调查" ──→ D2
    ├─ 包含"介绍/概述/整理/总结" ──→ D1
    ├─ 包含"看看/查一下/了解/快速" ──→ D0
    └─ 无明确关键词 ──→ 复杂度自动判定
                        ├─ 涉及学科数 ≤1 → D1
                        ├─ 涉及学科数 2-3 → D2
                        ├─ 涉及学科数 >3 → D3+
                        ├─ 时效要求高 → 降一级
                        ├─ 决策影响大 → 升一级
                        └─ 存在争议性 → 升一级
```

**深度升降级规则**:

| 触发条件 | 升/降级 | 调整幅度 |
|---------|---------|---------|
| 研究过程中发现重大矛盾/争议 | 升级 | +1~2级 |
| 发现高价值但需更深研究 | 升级 | +1级 |
| 资源/时间不足以支撑当前深度 | 降级 | -1级 |
| 决策紧迫性增加 | 降级 | -1级 |
| 数据质量不足以支撑当前深度 | 降级 | -1级 |

**动态调整触发规则**:

```
触发条件 → 调整动作:
  样本偏差 → 增加分层抽样/配对设计
  时间效应 → 增加纵向设计/时间序列分析
  群体差异 → 增加亚组分析/调节效应检验
  因果需求 → 增加实验设计/工具变量
  外部效度低 → 增加跨情境/跨文化验证
  内部效度低 → 增加控制变量/匹配设计
  数据质量差 → 增加清洗步骤/稳健性检验
```

### 1.5 跨层级组合规范

**组合原则**:

1. 互补原则: 各层级覆盖不同维度
2. 时序原则: 先定性探索后定量验证 / 先定量发现后定性解释
3. 嵌套原则: 大框架用低精度层级，关键子问题用高精度层级
4. 成本原则: 在满足精度需求的前提下，尽量使用低成本层级

**标准组合模式**:

| 组合模式 | 结构 | 适用场景 | 协同系数 | 示例 |
|---------|------|---------|---------|------|
| 探索-验证 | L2先行→L1验证 | 新领域研究 | 1.5x | 先深度访谈了解需求，再问卷量化验证 |
| 广度-深度 | L1广度→L2深度 | 大规模精细化 | 1.4x | 先问卷调查发现群体差异，再访谈深挖 |
| 监测-解释 | L3监测→L2解释 | 异常分析 | 1.3x | 数字化工具发现异常，定性研究解释原因 |
| 行为-心理 | L3行为→L4心理 | 深度用户理解 | 1.6x | 行为数据追踪+神经测量揭示潜意识 |
| 理论-实证 | L2理论→L1实证 | 模式验证 | 1.5x | 扎根理论构建模型+大样本统计验证 |
| 全景组合 | L1+L2+L3并行 | 战略级研究 | 2.0x | 定量+定性+数字化全面覆盖 |
| 终极组合 | L1+L2+L3+L4 | 重大决策 | 2.5x | 四级全覆盖，最高可信度 |

**深度-层级-组合三维决策**:

```
研究需求
    ↓
精度需求评估
    ├─ 低精度（快速决策）──→ 单层级 + D0-D1
    ├─ 中精度（常规决策）──→ 单层级 + D2-D3
    ├─ 高精度（重大决策）──→ 双层级组合 + D3-D4
    └─ 极高精度（战略决策）──→ 多层级组合 + D4-D5
```

### 1.6 三层质量门控体系

**Gate 1: 输入门控（研究前）**

- [ ] 研究问题表述清晰、可回答
- [ ] 层级选择有充分理由
- [ ] 深度级别与研究目标匹配
- [ ] 方法选择与研究问题匹配
- [ ] 样本/数据规模满足最低要求
- [ ] 时间/预算在可行范围内
- [ ] 伦理问题已评估（如涉及人类被试）

**Gate 2: 过程门控（研究中）**

- [ ] 数据收集按计划执行
- [ ] 数据质量检查通过（完整率/准确率/偏差度）
- [ ] 分析方法正确执行
- [ ] 异常值/缺失值处理有记录
- [ ] 中期发现已评估（是否需要调整）
- [ ] 进度在预期范围内

**Gate 3: 输出门控（研究后）**

- [ ] 结论有充分数据支撑
- [ ] 推断不超出数据边界
- [ ] 置信度/不确定性已标注
- [ ] 局限性充分说明
- [ ] 产出格式符合应用需求
- [ ] 可操作性建议具体可行
- [ ] 后续研究方向已列出

**各深度级别的质量标准**:

| 质量维度 | D0 | D1 | D2 | D3 | D4 | D5 |
|---------|-----|-----|-----|-----|-----|-----|
| 信息源可信度 | ≥1个权威源 | ≥2个可信源 | ≥3个交叉验证 | 权威文献为主 | 学术文献为主 | 全量文献覆盖 |
| 数据完整性 | 关键数据点 | 主要维度 | 关键指标完整 | 全维度覆盖 | 变量完备 | 全量+元数据 |
| 分析严谨度 | 描述性 | 基础分析 | 统计检验 | 多角度分析 | 理论框架驱动 | 逻辑严密+对抗检验 |
| 偏差控制 | 无要求 | 注意明显偏差 | 控制主要偏差 | 系统性偏差分析 | 全面偏差控制 | 偏差博弈分析 |
| 可复制性 | 无要求 | 列出信息源 | 方案可描述 | 方案可复制 | 方案可复现 | 完全可复现 |

**质量不通过的处理**:

```
质量检查未通过
    ↓
定位问题环节（输入/过程/输出）
    ├─ 输入问题 → 返回重新定义研究设计
    ├─ 过程问题 → 补充数据/调整方法
    ├─ 输出问题 → 调整结论范围/补充分析
    └─ 系统性问题 → 降级深度或升级方法
```

### 1.7 AI增强研究体系

**AI辅助研究全链路**:

| 研究环节 | AI角色 | AI能力 |

```

综合消费成本(t) = 金钱成本 + 时间成本 + 认知成本 + 机会成本 + 神经努力成本
```

## 第三层：价值量化与数据驱动

### 3.1 价值发现→数据维度映射框架

当研究发现一个潜在价值时，需要系统性地回答：

| 问题 | 说明 | 评估要点 |
|------|------|---------|
| 这是什么价值? | 描述发现的价值本质 | 价值类型分类 |
| 价值体现在哪里? | 用户行为? 商业结果? 体验提升? | 价值表现 |
| 如何量化这个价值? | 需要什么数据来证明? | 量化指标 |
| 价值的影响范围? | 影响哪些用户? 哪些场景? | 影响范围 |
| 价值的时效性? | 是短期效应还是长期价值? | 生命周期 |
| 价值的ROI是多少? | 投入产出比如何? | 投资回报 |
| 价值的置信度? | 这个发现有多可靠? | 置信评估 |

### 3.2 三维价值量化评估模型

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   成本价值   │    │   管理价值   │    │   应用价值   │
├─────────────┤    ├─────────────┤    ├─────────────┤
│ • 数据获取成本│    │ • 数据质量   │    │ • 决策支持   │
│ • 分析成本   │    │ • 可得性     │    │ • 效率提升   │
│ • 维护成本   │    │ • 透明度     │    │ • 业务增长   │
│ • 机会成本   │    │ • 可信度     │    │ • 用户价值   │
└─────────────┘    └─────────────┘    └─────────────┘

综合价值 = 0.2×成本价值 + 0.3×管理价值 + 0.5×应用价值
```

**ROI计算公式**:

```
基础ROI = 产出价值 / max(投入成本, 0.1)
效率评分 = 质量 × (1 + 节省时间/max(实际时间, 1)) × (0.9 if 已采纳 else 0.5)
置信度 = min(1.0, max(平均置信度, 样本量/max(1.0, 最低记录数)))
综合ROI = 0.2×成本价值 + 0.3×管理价值 + 0.5×应用价值
```

```

```

### 5.6 ICE评分与优先级

**ICE评分模型**:

| 维度 | 权重 | 评分标准（1-10分） |
|------|------|-------------------|
| Impact（影响） | 40% | 对目标指标的预期提升程度 |
| Confidence（置信） | 30% | 研究发现的可信度、数据质量 |
| Ease（难易） | 30% | 实施难度、资源需求、时间成本 |

综合评分 = Impact×0.4 + Confidence×0.3 + Ease×0.3

| 评分区间 | 优先级 | 决策建议 |
|---------|--------|----------|
| 8-10分 | P0 立即执行 | 资源倾斜，快速验证 |
| 6-8分 | P1 高优先级 | 排入近期计划 |
| 4-6分 | P2 中优先级 | 排入正常迭代 |
| 2-4分 | P3 低优先级 | 资源充裕时执行 |
| 0-2分 | P4 暂缓 | 暂不投入，记录观察 |

### 5.7 策略关联网络

**关系类型**:

```
策略A ──协同──→ 策略B  (同目标不同机制，同时使用效果更好)
策略A ──冲突──→ 策略B  (同维度完全重叠，需评估是否冲突)
策略A ──前置──→ 策略B  (策略A是策略B的前提)
策略A ──替代──→ 策略B  (策略A可以替代策略B)
策略A ──组合──→ 策略B  (同机制不同目标，可形成组合策略)
```

**自动检测规则**:

- 同目标 + 不同机制 → 协同 (strength=0.6)
- 同机制 + 不同目标 → 组合 (strength=0.5)
- 同目标 + 同机制 + 同环节 → 冲突 (strength=0.7)

**典型协同模式**:

| 协同模式 | 说明 | 示例 |
|---------|------|------|
| 漏斗协同 | 前段策略获客 + 后段策略转化 | 情绪内容增长 + 情绪触发转化 |
| 强化协同 | 主策略 + 强化辅助 | 情绪触发转化 + 紧迫情绪强化 |
| 留存协同 | 留存策略 + 激活策略 | 情绪维系留存 + 习惯养成激活 |
| 全链路协同 | 增长 + 转化 + 留存 + 体验 | 完整用户旅程覆盖 |

### 5.13 策略冲突处理决策

| 冲突场景 | 判断标准 | 处理方式 |
|---------|---------|---------|
| 同目标+同机制+同环节 | 完全重叠 | 选择效果更好的，另一个降级 |

### 5.15 常见问题速查

**Q1: 如何判断一个发现是否有价值？**

从三个维度判断：价值规模（影响多少用户/多少营收）、价值强度（效果有多显著）、价值可持续性（短期还是长期）。三维量化评估总分≥10分即可认为有价值。

**Q2: 一个发现可能有多个价值类型，怎么办？**

按价值规模排序，选择最大价值类型作为主策略类型，其他价值类型记录为附加价值。一个发现最多生成3条关联策略，按优先级排序。

**Q3: ICE评分差异大时如何决策？**

当Impact与Ease差异>4分时，优先看Confidence维度：Confidence高→信任高分区评分，倾向执行；Confidence低→需补充数据验证后再决策。

**Q4: 策略验证失败怎么办？**

区分失败原因：设计问题→调整策略设计；执行问题→改善执行方案；假设失效→退回发现期重新评估；外部变化→更新环境假设后重新验证。

**Q5: 如何发现策略之间的关联？**

使用自动检测规则：同目标+不同机制→协同、同机制+不同目标→组合、同目标+同机制+同环节→冲突。同时关注V5调度路径是否共享主线。

**Q6: 策略进入衰退期后如何处理？**

三步决策：效果下降<20%→优化参数继续执行；效果下降20-40%→启动替代策略生成；效果下降>40%或连续2个周期无改善→启动退役流程，知识沉淀归档。
