## 3. 双核心包架构

Somn 的可执行能力集中在两个 Python 包中：

```
┌─────────────────────────────────────────────────────────────────┐
│                    smart_office_assistant                        │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │                      src/                               │    │
│  │  core → intelligence → neural_memory → learning         │    │
│  │                    ↕                                     │    │
│  │              main_chain（主线调度）                      │    │
│  └─────────────────────────────────────────────────────────┘    │
│  __init__.py → sys.modules['src'] = sys.modules[__name__]       │
└─────────────────────────────────────────────────────────────────┘
                              ↕ 双向调用
┌─────────────────────────────────────────────────────────────────┐
│                      knowledge_cells/                           │
│  core.py → EightLayerPipeline → DomainNexus → PanWisdomTree    │
│  RefuteCore v3.1.0（预加载+懒加载）、DivineReason v4.0          │
│  WisdomDispatcher、神行轨 B 轨 Adapter                           │
└─────────────────────────────────────────────────────────────────┘
```

**核心架构机制（sys.modules 路径映射）**：
```python
# smart_office_assistant/__init__.py 和 src/__init__.py 中
import sys
sys.modules['src'] = sys.modules[__name__]
```
这使得 `from src.xxx import yyy` 与 `from smart_office_assistant.src.xxx import yyy` 等价，是全项目 700+ 条 `from src.` 导入能正常工作的基础。

## 10. 关键技术机制说明

### 10.1 sys.modules 路径映射（根治 from src. 导入问题）

```python
# 位置：smart_office_assistant/__init__.py 和 src/__init__.py
import sys
sys.modules['src'] = sys.modules[__name__]
```
效果：`from src.intelligence import X` 等价于 `from smart_office_assistant.src.intelligence import X`，解决了项目中 703+ 条 `from src.` 导入在包外使用时的 ModuleNotFoundError 问题。

### 10.2 四级懒加载体系（毫秒级启动）

```
L0 立即加载（<0.1ms）：core.py 基础类定义
L1 预加载（<3ms）：正则表达式、关键词库、注册表
L2 懒加载（首次~5ms）：调度器实例（首次使用时创建）
L3 按需加载（首次~5s）：763个Claw YAML（I/O密集，不可避免）
```

优化结果：
- 系统初始化：0.02ms（优化前 ~640ms，提升 32000x）
- 调度器加载：0.3ms → 缓存后 0.02ms
- 完整任务调度：4~8ms

### 10.3 延迟导出（`__getattr__`）

全项目所有 `__init__.py` 均使用 `__getattr__` 实现延迟导出：
```python
def __getattr__(name: str):
    if name == "Somn":
        from .somn import Somn
        return Somn
    raise AttributeError(f"module has no attribute {name!r}")
```
保证 `import smart_office_assistant` 不会触发链式加载。

### 10.4 双版本并行 smart\_office\_assitant

项目中存在两个目录：
- `smart_office_assistant/`（正确拼写，当前主包）
- `smart_office_assitant/`（拼写错误，旧版目录，354+ 条导入仍指向它）

旧版目录仍有 22 个文件、354+ 条 `from smart_office_assitant.` 导入，是历史遗留，建议逐步迁移。
