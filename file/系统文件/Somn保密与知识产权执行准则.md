# Somn 保密与知识产权执行准则

**生效日期**：2026-04-02  
**适用范围**：`smart_office_assistant/` 全部代码、配置、数据、文档、测试文件、研究资料、工作产物与派生内容  
**级别**：最高优先级硬约束

---

## 一、目标

本准则用于把 Somn 项目的保密、知识产权与本地运行边界固化为执行级规则。

核心原则只有一句：

> **项目内容只属于本地、属于你，不属于任何第三方、任何云端、任何外部协作面。**

---

## 二、执行级红线

### 1. 本地唯一源原则

- 项目的唯一可信源为本地项目文件夹。
- 代码、配置、数据、文档、测试样本、研究记录，均以本地文件为准。
- 每次打开、启动、调用、调试本项目时，只能从本地项目目录加载，不得从任何远程仓库、云端镜像、在线沙箱、云盘挂载目录或网页缓存副本启动。

### 2. 禁止上传云端

严禁将以下内容上传到任何网络环境或云端服务：

- 源代码
- 配置文件
- 提示词与系统设计
- 测试数据与日志
- 研究文档、分析笔记、阶段性结论
- 任意片段、截图、摘要、样例代码

禁止对象包括但不限于：

- GitHub / Gitee / GitLab / Bitbucket
- 云存储 / 网盘 / 对象存储
- 在线代码托管平台
- 云函数、在线 Notebook、在线 IDE
- 任何外部协作系统、自动同步工具、远程缓存服务

### 3. 禁止云端缓存与留痕

除了禁止上传，还禁止以任何形式在云端留下缓存、索引、镜像或副本，包括但不限于：

- 自动同步生成的副本
- 云端备份与历史版本
- 在线编辑器缓存
- 第三方插件日志或遥测副本
- 远程构建产物、调试快照、临时转储

**要求**：即使不是“正式上传”，只要会在云端留下项目痕迹，也视为违规。

### 4. 本地启动与本地调用原则

- 项目必须从本地目录直接启动。
- 项目运行时读取的代码、配置、数据，应来自本地项目路径。
- 不允许把项目复制到云工作区后再运行。
- 不允许通过在线代理、远程容器、云端脚本执行器调用本项目代码。
- 如需临时文件、缓存或导出文件，也必须落在本机本地路径，不得落到任何同步盘或云挂载目录。

### 5. 对外信息最小化原则

- 查阅公开资料时，只允许读取公开信息。
- 不得把项目私有内容作为查询材料发送到外部环境。
- 不得向任何第三方披露你正在研究什么、做到哪一步、采用了什么方法，哪怕只是一句代码或一段注释。

### 6. 本地版本控制原则

- 可以使用本地 Git 仓库。
- 禁止配置、绑定或推送到任何远程仓库。
- 禁止使用会自动同步到云端的版本控制托管服务。

### 7. 对外动作先确认原则

凡是满足以下任一条件，均必须先获得你的明确许可：

- 会离开本地文件系统
- 会接触第三方服务
- 会创建外部可见副本
- 会发送项目文本、代码、配置、截图、日志

未获明确许可，默认不执行。

---

## 三、开发与运行要求

### 1. 路径要求

- 使用绝对路径优先。
- 默认工作根目录应为本地 `smart_office_assistant/` 项目目录。
- 禁止从未知路径、网络映射路径、云同步路径引入运行依赖。

### 2. 联网要求

- 联网仅用于查询公开资料。
- 联网请求不得携带项目代码、私有文档、研究记录、用户数据、测试数据。
- 若某操作天然需要上传内容，默认禁止，除非你明确批准。

### 3. 日志要求

- 运行日志仅保留在本地。
- 日志中避免写入完整敏感内容；必要时做脱敏处理。
- 不启用会将日志、错误栈、遥测信息自动发往外部的功能。

### 4. 缓存要求

- 缓存仅允许存在于本机本地目录。
- 禁止使用带自动同步、自动备份、在线索引能力的缓存位置。
- 清理缓存时，优先清理本地副本，不创建远端归档。

---

## 四、知识产权归属与处理原则

- 项目代码、架构、文档、方法论、研究路径与派生成果，默认视为受保护资产。
- 未经你的明确授权，不得对外复用、展示、传播、共享、转载或训练到任何外部系统。
- 任何整理、摘要、改写、示例化后的内容，只要源于本项目，仍按项目私有资产处理。

---

## 五、执行检查清单

每次开展开发、调试、整理、运行前，默认检查：

1. 当前工作目录是否为本地项目目录？
2. 本次读取/写入是否全部发生在本地？
3. 是否存在云盘、同步盘、在线 IDE、远程容器路径？
4. 是否存在任何上传、同步、遥测、远程缓存风险？
5. 是否有任何内容将离开本地边界？

只要任意一项答案不安全，立即停止该动作。

---

## 六、后续技术固化建议

为把本准则从“约定”升级为“机制”，后续建议补上以下能力：

- 本地项目根目录启动校验器
- 云同步/远程路径探测器
- 外发动作拦截器
- 运行时本地缓存目录白名单
- 敏感内容外传审计开关

---

## 七、结论

Somn 的建设目标可以很高，甚至朝超级智能体推进；
但它的运行边界必须更硬：

> **能力可以升级，边界不能松动。**  
> **智能可以增强，项目必须留在本地。**
