## 三、问题根因分析

### 3.1 可变全局常量（254个）

**典型模式**:
```python
# 模式A：临时变量在函数外定义
def process():
    results = []  # 应该在函数内
    for item in items:
        results.append(item)
    return results

# 模式B：缓存字典
_cache = {}  # 应该用专门的缓存机制

# 模式C：计数器
_counter = []  # 应该用类封装
```

**根因**: 早期快速开发时，为简化作用域管理，将临时变量外提。

**风险**: 多线程/异步环境下可能产生竞态条件，状态污染难以追踪。

### 3.2 多入口点（100个）

**典型模式**:
```python
# src/some_module.py
def main():
    print("Running")

if __name__ == "__main__":
    main()
```

**根因**: 模块既作为库被导入，又作为独立工具运行。

**风险**: 入口不唯一导致执行路径不确定，测试覆盖困难。

### 3.3 缺少 `__all__` 导出（522个）

**典型模式**:
```python
# src/my_module.py
def public_func():
    pass

def _private_func():
    pass

# 没有 __all__，导致 from src.my_module import * 导入所有内容
```

**根因**: 未遵循模块接口封装规范。

**风险**: 公共API与内部实现混淆，模块重构困难。
