## 零、框架层与实现层 — 双重视角

Somn 神之架构存在两个观察视角，**框架层**是**实现层**的抽象描述，两者一体两面。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          框架层 Framework                               │
│                                                                         │
│     输入 ──→ 智慧矩阵 ──→ 推理矩阵 ──→ 论证体系 ──→ 输出              │
│                                                                         │
│     【视角】: 外部观察者视角，描述"做什么"                               │
│     【特点】: 概念清晰，便于沟通；不影响具体实现                         │
│     【定位】: 对天枢的侧面描述，高层次抽象                               │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
                                    │ 映射
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                          实现层 Implementation                          │
│                                                                         │
│  L1输入 → L2NLP → L3分类 → L4分流 → L5推理 → L6论证 → L7输出 → L8优化 │
│                                                                         │
│     【视角】: 代码实现视角，描述"怎么做"                                 │
│     【特点】: 技术完整，可执行验证                                       │
│     【定位】: 天枢的完整代码实现，承载框架层所有能力                     │
└─────────────────────────────────────────────────────────────────────────┘
```

### 框架层 → 实现层 映射表

| 框架层 | 实现层 | 说明 |
|--------|--------|------|
| **输入** | L1 输入层 | 问题接收、格式标准化、意图识别 |
| **智慧矩阵** | L3 分类 + DivineReason + Pan-Wisdom | 问题类型匹配 × 42学派融合 |
| **推理矩阵** | L5 推理层 + DomainNexus | 深度推理 + 知识库增强 |
| **论证体系** | L6 论证层 + RefuteCore + SD-R2 | 谬误检测 + 8维度反驳 |
| **输出** | L7 输出层 + L8 优化层 | 结果整合 + 质量优化 |

### 重要原则

> **框架层不影响实现层**
> - 框架层是对天枢的**侧面描述**，用于沟通和理解
> - 天枢的具体流程、调度逻辑、技术实现**保持不变**
> - 框架层与实现层一一对应，但不互相侵入

## 二、核心模块详解

### 2.1 天枢 TianShu — 八层智慧处理管道

**定位**: 系统运转中枢，统御八层智慧，协调所有模块协作

| 层级 | 名称 | 核心功能 | 关键模块 |
|------|------|----------|----------|
| L1 | 输入层 | 问题接收、格式标准化、意图识别 | InputLayer |
| L2 | NLP分析层 | 语义理解、关键词提取、上下文分析 | NLAnalysisLayer |
| L3 | 需求分类层 | 问题类型匹配、紧急度评估 | ClassificationDB |
| L4 | 分流层 | 调度路径选择、资源分配、策略制定 | RoutingLayer |
| L5 | 推理层 | 智慧融合、深度推理、知识调用 | ReasoningLayer + DomainNexus |
| L6 | 论证层 | 论证生成、质量评估、谬误检测 | ArgumentationLayer + RefuteCore |
| L7 | 输出层 | 结果整合、格式化输出、置信度评估 | OutputLayer |
| L8 | 优化层 | 冗余消除、语言优化、一致性检查 | OptimizationLayer |

**三等级处理**:
- **Basic**: P1→F2→E1 快速响应，不进神之架构
- **Deep**: P1→F2→C2→E1 神之架构并行+Claw讨论
- **Super**: P1→F1+F2→C2→E1 全链路Claw+T2 RefuteCore

**论证闭环回退机制**: L6论证不合格 → 返回L4重新分流 → 最大重试3次

### 2.2 DivineReason V4.0 — 三层推理引擎

**定位**: 系统核心推理能力，提供表层→深度→元认知三层推理

| 等级 | 调度器 | 推理深度 | 适用场景 | 性能目标 |
|------|--------|----------|----------|----------|
| Light | SD-D1 | Layer1 表层推理 - 模式匹配 | 快速问答、简单查询 | <1s |
| Standard | SD-D2 | Layer1+2 深度推理 - 假设探索 | 复杂分析、策略制定 | 5-10s |
| Deep | SD-D3 | Layer1+2+3 极致推理 - 元认知审视 | 战略决策、创新突破 | 10-30s |

**Layer层说明**:
- **Layer1 表层推理**: 模式识别、关键词匹配、基础逻辑
- **Layer2 深度推理**: 假设探索、多角度分析、证据评估
- **Layer3 元认知**: 推理过程审视、认知偏差检测、思维优化

### 2.5 双轨系统 — 神政轨(A) + 神行轨(B)

**A轨神政轨**: 思考决策层 — 军师/宰相角色
- 接收用户请求
- 问题分析与战略制定 (DivineReason + Pan-Wisdom)
- 派发任务给B轨
- 验收执行结果
- 全局监管 (SD-R1三层审查)

**B轨神行轨**: 执行落地层 — 牛马/干将角色
- 接收A轨任务
- 分配给11部门执行器
- 调动763个Claw执行
- 回报执行结果给A轨

**11部门执行器**:

| 部门 | 职责 | 对接调度器 |

| 吏部 | 人才评估、团队建设 | SD-D系列 |
| 户部 | 资源配置、财务规划 | SD-L1 |
| 礼部 | 文化融合、跨学派智慧 | SD-F1 |
| 刑部 | 合规审查、风险控制 | SD-R2 |
| 工部 | 执行落地、项目管理 | SD-E1 |
| 外交部 | 外联合作、公共关系 | - |
| 教育部 | 培训发展、能力提升 | - |
| 科技部 | 技术创新、研发管理 | - |
| 文化部 | 品牌传播、内容营销 | - |
| 卫生部 | 健康管理、生活品质 | - |

**特权调用机制**:

| 调用方 | 调用权限 | 调用方式 |
|--------|----------|----------|
| DivineReason(节点) | ✅ 可直接调用B轨 | 跳过调度器 |
| Pan-Wisdom(枝丫) | ✅ 可直接调用B轨 | 跳过调度器 |
| 神政轨 | ✅ 可通过桥接器调用 | TrackBridge |
| 其他模块 | ❌ 禁止直接调用 | 需通过调度器 |

### 2.6 DomainNexus v2.2 — 知识域格子系统

**定位**: 动态知识库，支持按需查询与自动进化

- **快速查询**: 预加载索引 + LRU缓存，<1ms响应
- **自动进化**: 使用情况追踪 → 缺口发现 → 智能丰富
- **绝对安全**: 禁止删除任何知识点
- **21知识格子**: 智慧核心(A1-A8) + 知识域(B1-C4)

---

### 2.7 NeuralMemory — 三层神经记忆系统

**定位**: 模拟人脑记忆机制，实现智能学习与遗忘

| 层级 | 名称 | 容量 | 衰减 | 进化目标 |
|------|------|------|------|----------|
| L1 | 短期记忆 | 7±2 chunks | 高频维持 | 重要→中期 |
| L2 | 中期记忆 | 100-1000事件 | 时间衰减 | 经验→长期 |
| L3 | 长期记忆 | 无限制 | 弱化连接 | 持续优化 |

---

### 2.8 RefuteCore v3.2 — 驳心引擎

**定位**: 8维度反驳论证系统，确保推理质量

| 维度 | 名称 | 检测内容 |
|------|------|----------|
| 1 | 逻辑漏洞 | 前提/推理链缺陷 |
| 2 | 证据不足 | 数据/引用可靠性 |
| 3 | 假设错误 | 隐含前提挖掘 |
| 4 | 反面案例 | 反例驳斥 |
| 5 | 类比失当 | 类比逻辑检查 |
| 6 | 权威依赖 | 来源可信度评估 |
| 7 | 因果混乱 | 相关/因果区分 |
| 8 | 价值偏差 | 主观倾向检测 |


强度等级: S/A/B/C/D/F | 风险判定: 🔴红/🟠橙/🟡黄/🟢绿
