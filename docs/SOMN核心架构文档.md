### 3.2 双轨系统

| 轨道 | 名称 | 角色 | 职责 |
|------|------|------|------|
| **A轨** | 神政轨 | 思考者/军师 | DivineReason推理、Pan-Wisdom融合、SageDispatch调度 |
| **B轨** | 神行轨 | 执行者/牛马 | 11部门×763Claw落地执行 |

**调用关系**: A轨 → B轨 (单向特权调用)

## 四、神之架构 vs SOMN核心架构

> **重要区分**

| 概念 | 范围 | 定位 |
|------|------|------|
| **SOMN核心架构** | 整个项目 | 完整系统架构，包含UI、API、Engine、Knowledge |
| **神之架构** | Knowledge Cells | 子系统架构，专注于智慧调度引擎 |

```
SOMN 核心架构 (整体)
├── Somn-GUI (UI层)
├── API Server (服务层)
├── Somn Engine (核心层)
│   ├── Dual Track (双轨系统)
│   └── NeuralMemory (三层记忆)
└── Knowledge Cells (知识层) ← 神之架构
    ├── 天枢 Pipeline (八层管道)
    ├── DivineReason (三层推理)
    ├── Pan-Wisdom (42学派)
    ├── DomainNexus (21格子)
    └── 神行轨 (11部门×763Claw)
```
